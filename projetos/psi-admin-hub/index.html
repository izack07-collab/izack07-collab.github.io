PSI Admin Hub

React + Vite TypeScript Shadcn/UI Supabase Admin Recharts RLS Policies

Painel interno de administração da plataforma PainelPsi. Enquanto o painel do psicólogo cuida do consultório, o Admin Hub dá controle total sobre a plataforma SaaS: aprovação e monitoramento de contas, analytics de uso e retenção, logs de auditoria e configurações globais — tudo em tempo real, protegido por RLS e autenticação de nível administrativo.

Módulos do Painel

📊
Dashboard de Plataforma
Visão macro em tempo real: total de psicólogos ativos, novos cadastros da semana, conversão de trial para pago e métricas de engajamento por feature.
👥
Gestão de Psicólogos
Lista completa de usuários com filtros de status (ativo, trial, suspenso). Aprovação e suspensão de contas com um clique, e acesso ao perfil detalhado de cada profissional.
📈
Analytics de Retenção
Gráficos de uso por feature (pacientes, agenda, financeiro), retenção semanal/mensal e identificação de usuários em risco de churn.
📋
Logs de Auditoria
Registro de todas as ações críticas da plataforma: login de admins, alterações de conta, aprovações e suspensões, com timestamps e rastreio por usuário.
⚙️
Configurações Globais
Controle centralizado de limites de trial (dias e pacientes), feature flags por plano, e configurações de notificações e webhooks para integrações externas.
🔒
Segurança & Acesso
Acesso restrito por role de admin no Supabase Auth. Políticas de RLS garantem que nenhum psicólogo possa acessar dados internos da plataforma.

Arquitetura Técnica

Frontend
React + Vite + TypeScript — SPA de alta performance com componentes Shadcn/UI e Recharts para visualização de dados.
Autenticação
Autenticação via Supabase Auth com verificação de role admin. Redirecionamento automático para login caso o usuário não tenha permissão administrativa.
Banco de Dados
PostgreSQL + Supabase — as mesmas tabelas do painel do psicólogo, acessadas via queries privilegiadas com service_role onde necessário.
Segurança
Row Level Security (RLS) com políticas separadas por role. Nenhum dado de paciente é exposto ao admin — apenas metadados de uso e contas de psicólogos.
