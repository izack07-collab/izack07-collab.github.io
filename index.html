Dashboard de BI · Efetividade de Campanhas WhatsApp

Em Produção

Dashboard de BI (Metabase) sobre MongoDB para monitorar a efetividade de campanhas de cobrança via WhatsApp. Construí uma pipeline de agregação com $lookup unindo o registro de disparo aos callbacks de status da API do WhatsApp, reduzindo o funil cumulativo de status (aceito → entregue → lido) a um único estado por mensagem, sem contagem duplicada. Validei cada número com consultas de agregação reproduzíveis antes de publicar, e identifiquei um achado operacional relevante: quase metade dos envios falhava antes de aceitos pela Meta, dividido entre limitação de taxa de disparo (infraestrutura) e número de telefone inválido (qualidade de dado). São dois problemas de responsável distinto, documentados com metodologia reproduzível. Também diagnostiquei e corrigi uma visualização existente que somava indevidamente as mesmas mensagens em múltiplas categorias, substituindo por um modelo de estado terminal único por mensagem.

Investigação orientada a dado que virou dashboard de produção e um achado de negócio acionável

MongoDB (Aggregation Framework) Metabase WhatsApp Cloud API mongosh
